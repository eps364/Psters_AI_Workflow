# Checklist de Qualidade de Documentacao

Use este checklist antes de fazer merge de mudancas em docs.

## Qualidade de conteudo

- [ ] Escopo explicito (o que mudou, onde, e por que).
- [ ] Instrucoes acionaveis e especificas.
- [ ] Terminologia consistente com o vocabulario do projeto.
- [ ] Uso correto de `/pwf-work`, `/pwf-work-plan` e familia `/pwf-doc` (`/pwf-doc`, `/pwf-doc-foundation`, `/pwf-doc-runbook`, `/pwf-doc-capture`).
- [ ] Nenhum dado confidencial interno foi exposto.

## Qualidade de estrutura

- [ ] Titulos claros e previsiveis.
- [ ] Secoes seguem ordem logica.
- [ ] Pontos criticos/caveats estao destacados.
- [ ] Links para docs relacionadas foram incluidos quando relevante.

## Consistencia bilingue

- [ ] Versoes EN e PT estao alinhadas semanticamente.
- [ ] Nomes de comandos permanecem iguais nas duas linguas.
- [ ] Significado preservado (nao traducao literal incorreta).

## Verificacoes tecnicas

- [ ] Markdown renderiza corretamente no preview do repositorio.
- [ ] Diagramas Mermaid (quando houver) renderizam corretamente.
- [ ] Links internos validos.
- [ ] Nao ha referencias obsoletas.
