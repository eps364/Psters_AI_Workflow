# Docs Quality Checklist

Use this checklist before merging documentation changes.

## Content quality

- [ ] Scope is explicit (what changed, where, and why).
- [ ] Instructions are actionable and specific.
- [ ] Terminology is consistent with project vocabulary.
- [ ] `/pwf-work`, `/pwf-work-plan`, and `/pwf-doc` family (`/pwf-doc`, `/pwf-doc-foundation`, `/pwf-doc-runbook`, `/pwf-doc-capture`) usage is correct.
- [ ] No project-internal confidential data is exposed.

## Structure quality

- [ ] Headings are clear and predictable.
- [ ] Sections follow a logical order.
- [ ] Important caveats are highlighted.
- [ ] Links to related docs are included when relevant.

## Bilingual consistency

- [ ] English and Portuguese versions are semantically aligned.
- [ ] Command names are unchanged across languages.
- [ ] Meaning is preserved (not literal but inaccurate translation).

## Technical checks

- [ ] Markdown renders correctly in repository preview.
- [ ] Mermaid diagrams (if any) render correctly.
- [ ] Internal links are valid.
- [ ] No stale references remain.
