# Contributing

Thank you for your interest in improving `psters-ai-workflow`.

This project is open to contributions from anyone who wants to:

- add new agents
- add or improve commands
- improve existing rules/skills/docs
- add language-specific or framework-specific workflow expansions
- improve developer experience for the whole community
- help at all

## How to contribute

### Option A: Idea-only contribution (no code required)

1. Open an issue in this GitHub repository.
2. Describe your idea clearly (problem, expected behavior, value).
3. Add context/examples if possible.
4. If you do not want to implement it, say so in the issue.

Someone from the community (or maintainers) can pick up and implement your idea.

### Option B: Code contribution

1. Open an issue describing your idea, problem, or proposal.
2. Discuss direction and scope in the issue.
3. Create a pull request with focused changes.
4. Include clear rationale and usage examples when relevant.
5. Link your PR to the issue.

We will review, discuss, and evaluate each proposal on GitHub.

## Review principles

Contributions are evaluated based on:

- clarity and practical usefulness
- consistency with anti-vibe-coding workflow principles
- quality, maintainability, and documentation impact
- community value and adoption potential

If the proposal makes sense and receives positive feedback, we can merge and evolve it together.

## Recognition

Everyone who contributes meaningfully will be recognized as a contributor in this project.

# Contribuindo

Obrigado pelo interesse em melhorar o `psters-ai-workflow`.

Este projeto e aberto para contribuicoes de qualquer pessoa que queira:

- adicionar novos agents
- adicionar ou melhorar comandos
- melhorar rules/skills/docs existentes
- criar expansoes especificas por linguagem ou framework
- melhorar a experiencia dos desenvolvedores em geral

## Como contribuir

### Opcao A: Contribuicao so com ideia (sem codigo)

1. Abra uma issue neste repositorio no GitHub.
2. Descreva sua ideia com clareza (problema, comportamento esperado, valor).
3. Adicione contexto/exemplos quando possivel.
4. Se nao quiser implementar, deixe isso explicito na issue.

Alguem da comunidade (ou mantenedores) pode assumir e implementar sua ideia.

### Opcao B: Contribuicao com codigo

1. Abra uma issue descrevendo sua ideia, problema ou proposta.
2. Discuta direcao e escopo na issue.
3. Crie um pull request com mudancas focadas.
4. Inclua racional claro e exemplos de uso quando fizer sentido.
5. Relacione o PR com a issue.

Vamos revisar, discutir e avaliar cada proposta no GitHub.

## Principios de avaliacao

As contribuicoes serao avaliadas por:

- clareza e utilidade pratica
- consistencia com os principios anti-vibe-coding
- qualidade, manutencao e impacto na documentacao
- valor para a comunidade e potencial de adocao

Se a proposta fizer sentido e tiver boa recepcao, podemos integrar e evoluir juntos.

## Reconhecimento

Toda pessoa que contribuir de forma relevante sera reconhecida como contributor neste projeto.
